# 🏹 궁기병 (Mounted Archer)

> **멈추지 않는 질주, 조선의 무예로 도깨비 무리를 토벌하라.**
> 

## 📋 Project Overview

- **Genre:** Top-down Action Shooter
- **Environment:** Unity 6 (6000.0.xf1)
- **Theme:** 조선 시대 마상 무예(Horse Archery) x 한국 전통 판타지(Dokkaebi)
- **Core Loop:** 강제 이동 제약 속에서 이동 방향과 공격 타이밍을 결정하는 긴박한 전투 경험

---

## ✨ Key Mechanics

- **Non-stop Action:** 플레이어는 멈추지 않고 전진합니다. 오로지 방향 제어만을 통해 적의 포위망을 돌파해야 합니다.
- **Power Charge:** 활 시위를 당기는 시간에 비례해 파괴력이 상승합니다. (최대 차징 시 **관통 효과** 및 **2.5배 데미지**)
- **Active Defense:**
    - **Intercept:** 날아오는 적의 투사체를 화살로 직접 맞추어 파괴할 수 있습니다.
    - **Guard/Reflection:** 환도를 사용하여 적의 공격을 방어하거나 반사하여 역습의 기회를 만듭니다.

---

## 🕹 Controls

활과 환도는 좌클릭 ↔ 우클릭으로 스왑할 수 있으며 대쉬와 더미 스킬은 원하는 키보드 키로 바꿀 수 있습니다.

### [Basic Controls]

| Key | Action | Description |
| --- | --- | --- |
| **WASD** | 이동 방향 | 캐릭터가 달리는 방향을 결정합니다. |
| **우클릭 (Hold)** | 시위 당기기 | 최대 1.5초간 차징 (강력한 관통 화살 준비) |
| **우클릭 (Release)** | 화살 발사 | 차징 시간에 비례한 강력한 공격 발사 |

### [Active Skills]

| Key | Action | Description |
| --- | --- | --- |
| **좌클릭** | 환도 사용 | **가드**(1초간 방어) 혹은 **반사**(투사체 튕겨내기) 수행 |
| **Space** | 대쉬 (Dash) | 현재 이동 방향으로 짧고 빠르게 이동 |
| **Shift** | 더미 (Decoy) | 마우스 위치에 더미 설치. 일정 시간 적의 어그로 유도 |

## 🛠️ Key Implementation (핵심 기술 구현)

이 프로젝트는 6주라는 짧은 기간 내에 핵심 재미를 완성하기 위해, **구조적 분리**와 **데이터 기반 설계**라는 두 가지 포인트에 집중했습니다.

### 1. 효율적인 플레이어 오브젝트 구조 (Hierarchy & Logic)

플레이어의 이동과 조준이 독립적으로 일어나는 '마상 슈팅'의 특징을 살리기 위해 오브젝트 계층을 분리하고 Unity LifeCycle을 최적화했습니다.

- **구조적 분리:**
    - `General` (전투/회전): 마우스 방향으로 독립적인 회전 및 애니메이션 처리.
    - `Horse` (이동 비주얼): 이동 속도와 애니메이션 재생 속도 동기화.
    - `Pointer` (피드백): `LateUpdate`를 활용해 캐릭터 이동 후의 최종 위치에 렌더링하여 지터링 방지 및 시각적 상태(차징, 쿨타임 등) 전달.
- **물리 및 로직 최적화:** `Update`에서 이동 속도를 계산하고, `FixedUpdate`에서 물리 연산을 수행하여 안정적인 프레임워크를 구축했습니다.
- **유연한 스킬 관리:** `PlayerSkillManager`를 통해 UI와 플레이어 데이터를 연결하여 스킬의 활성화 상태를 실시간으로 갱신하도록 설계했습니다.

### 2. 데이터 기반 스테이지 시스템 (Data-Driven Stage System)

20개의 스테이지를 효율적으로 관리하고 난이도 밸런싱을 용이하게 하기 위해 데이터 중심의 스폰 시스템을 구축했습니다.

- **확장성 있는 데이터 구조:** 스테이지 정보를 하드코딩하지 않고 `StageInfo`, `WaveData`, `EnemySpawnData` 등의 클래스로 구조화하여 관리합니다.
- **스폰 로직의 분리:**
    - `SpawnManager`: 실제 몬스터의 생성과 개수 제한(`spawnLimit`) 등 런타임 로직 담당.
    - `StageManager`: 스테이지별 몬스터 구성, 확률(`weight`) 등 데이터 참조 담당.
- **설계 의도:** 몬스터 종류의 추가나 스테이지 난이도 조절이 빈번할 것을 예측하여, 코드 수정 없이 데이터 필드 수정만으로 밸런스를 조절할 수 있는 구조를 지향했습니다.
